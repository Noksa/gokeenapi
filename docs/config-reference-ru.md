## `keenetic` — Подключение к роутеру

Обязателен для каждой команды.

| Поле | Тип | Обязательно | По умолчанию | Описание |
|---|---|---|---|---|
| `url` | string | ✅ | — | URL роутера. Принимает IP адрес или KeenDNS имя хоста. Поддерживает `http://` и `https://`. Пример: `http://192.168.1.1` |
| `login` | string | ✅ | — | Имя пользователя администратора роутера. Переопределяется переменной окружения `GOKEENAPI_KEENETIC_LOGIN`. |
| `password` | string | ✅ | — | Пароль администратора роутера. Переопределяется переменной окружения `GOKEENAPI_KEENETIC_PASSWORD`. |
| `tls_skip_verify` | bool | ❌ | `false` | Отключить проверку TLS сертификата. Включите, когда роутер использует самоподписанный сертификат. |

## `routes` — Статические маршруты

Используется командами: `add-routes`, `delete-routes`.

Список конфигураций маршрутизации для каждого интерфейса.

| Поле | Тип | Обязательно | Описание |
|---|---|---|---|
| `interfaceId` | string | ✅ | ID целевого интерфейса (например, `Wireguard0`). Запустите `show-interfaces` для просмотра доступных ID. |
| `bat-file` | список строк | ❌ | Пути к локальным `.bat` файлам с командами `route add`. Путь `.yaml`/`.yml` раскрывается в содержащийся в нём список `bat-file`. Относительные пути разрешаются относительно директории конфигурационного файла. |
| `bat-url` | список строк | ❌ | Удалённые URL с `.bat` файлами. Путь `.yaml`/`.yml` раскрывается в содержащийся в нём список `bat-url`. |

Для каждой записи должно быть указано хотя бы одно из `bat-file` или `bat-url`.

## `dns.records` — Статические DNS записи

Используется командами: `add-dns-records`, `delete-dns-records`.

Список статических DNS записей для создания в локальном резолвере роутера. Каждая запись сопоставляет одно доменное имя с одним или несколькими IPv4 адресами.

| Поле | Тип | Обязательно | Описание |
|---|---|---|---|
| `domain` | string | ✅ | Доменное имя для разрешения (например, `myserver.local`). |
| `ip` | список строк | ✅ | Один или несколько IPv4 адресов, которым должен соответствовать домен. |

Пример:

```yaml
dns:
  records:
    - domain: myserver.local
      ip:
        - 192.168.1.100
        - 192.168.1.101
    - domain: db.local
      ip:
        - 10.0.0.5
```

## `dns.routes.groups` — Группы DNS-маршрутизации

Используется командами: `add-dns-routing`, `delete-dns-routing`.

Список доменных групп для маршрутизации на основе политик. Каждая группа создаёт object-group и dns-proxy маршрут на роутере. Требуется прошивка Keenetic версии ≥ 5.0.1.

Каждая запись — это либо **объект группы**, либо **путь к `.yaml` файлу** содержащему список `groups:` (для использования общих групп в конфигурациях нескольких роутеров).

**Поля объекта группы:**

| Поле | Тип | Обязательно | Описание |
|---|---|---|---|
| `name` | string | ✅ | Уникальное имя объекта-группы на роутере. |
| `interfaceId` | string | ✅ | Целевой интерфейс для маршрутизации трафика совпавших доменов (например, `Wireguard0`). Запустите `show-interfaces` для просмотра доступных ID. |
| `domain-file` | список строк | ❌ | Пути к локальным `.txt` файлам с одним доменом на строку (строки, начинающиеся с `#`, являются комментариями). Путь `.yaml`/`.yml` раскрывается в содержащийся в нём список `domain-file`. |
| `domain-url` | список строк | ❌ | Удалённые URL со списками доменов (один домен на строку). Путь `.yaml`/`.yml` раскрывается в содержащийся в нём список `domain-url`. |

Для каждой группы обязательно должно быть указано хотя бы одно из `domain-file` или `domain-url`.

Пример:

```yaml
dns:
  routes:
    groups:
      - name: streaming
        domain-file:
          - domains/netflix.txt
          - domains/youtube.txt
        domain-url:
          - https://example.com/spotify-domains.txt
        interfaceId: Wireguard0

      - name: work-vpn
        domain-file:
          - domains/internal.txt
        interfaceId: GigabitEthernet0/Vlan4

      # Импорт общих групп из файла
      - common/shared_groups.yaml
```

## `add-awg` / `update-awg` — Команды WireGuard

Эти команды не читают специфичный раздел из конфигурационного файла. Им нужен только блок подключения `keenetic`. Конфигурация WireGuard передаётся через флаг CLI `--conf-file`, указывающий на стандартный файл WireGuard `.conf`:

```ini
[Interface]
PrivateKey = <private-key>
Address = 10.0.0.2/32

[Peer]
PublicKey = <public-key>
Endpoint = vpn.example.com:51820
AllowedIPs = 0.0.0.0/0
```

| Флаг CLI | Обязательно | Описание |
|---|---|---|
| `--conf-file` | ✅ | Путь к файлу WireGuard `.conf`. |
| `--name` | ❌ (только `add-awg`) | Имя нового интерфейса. Генерируется автоматически, если не указано. |
| `--interface-id` | ✅ (только `update-awg`) | ID существующего интерфейса для обновления. |

| Поле | Тип | Обязательно | По умолчанию | Описание |
